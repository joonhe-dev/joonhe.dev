## Introduction

In 2024, I quit my big-tech job and started building my own SaaS product as a one-person company. The first month, I spent two full weeks agonizing over the tech stack — Next.js or Nuxt? Supabase or PocketBase? Vercel or AWS?

Two weeks later I realized: **your tech stack doesn't determine product success, but the wrong one will waste a ton of your time.**

The core tension of a solo company is: **you're one person, but you need to cover frontend, backend, DevOps, payments, email, domains, and everything else.** Every minute spent "fighting infrastructure" is a minute stolen from building the product and acquiring users.

This article is the sum of my lessons learned over the past year. Hope it saves you some detours.

## Core Principles for a Solo Tech Stack

Before diving into specific choices, let's nail down a few principles:

| Principle | What It Means | Anti-Pattern |
|-----------|---------------|--------------|
| Low-cost start | Keep monthly spend at $0–$50; upgrade after product validation | Buying a $200/mo enterprise plan on day one |
| High-leverage tools | One tool solves multiple problems (e.g., Supabase handles DB + Auth + Storage) | Using a separate service for every feature |
| Fast iteration | From idea to launch in under 2 weeks | Spending 3 months building infrastructure |
| Portability | Don't get locked into a platform; be ready to migrate | Using a niche PaaS's proprietary API |
| Active community | Problems you hit should be Google-able | Picking a framework with 100 stars on GitHub |

**TL;DR: Pick tools that let you focus on writing business code, not infrastructure code.**

## Frontend Framework: Next.js Is the Default Choice

### Why Next.js?

For a one-person company, Next.js is almost the only reasonable choice:

```typescript
// app/page.tsx — a typical Next.js App Router page
import { createServerComponentClient } from '@supabase/auth-helpers-nextjs'
import { cookies } from 'next/headers'

export default async function DashboardPage() {
  const supabase = createServerComponentClient({ cookies })
  const { data: projects } = await supabase.from('projects').select('*')

  return (
    <div className="grid grid-cols-1 md:grid-cols-3 gap-4">
      {projects?.map((project) => (
        <ProjectCard key={project.id} project={project} />
      ))}
    </div>
  )
}
```

**Core advantages:**

1. **Full-stack capability** — API Routes, Server Components, Server Actions — one person handles frontend and backend
2. **Dead-simple deployment** — `git push` to Vercel and you're live, zero DevOps
3. **Mature ecosystem** — there's a ready-made solution for every problem
4. **SEO-friendly** — SSR/SSG out of the box

### When to Choose Nuxt?

If you're more comfortable in the Vue ecosystem, Nuxt 3 is a solid choice. But consider:

- Next.js's community is 5–10x the size of Nuxt's
- Vercel optimizes for Next.js like it's their firstborn
- The React Server Components ecosystem is exploding

**My take: unless you're already a heavy Vue user, go with Next.js.**

## Backend & Database: Supabase Is the Best Bang for Your Buck

### Why Supabase Works for Solo Founders

Supabase solves the biggest headache of building SaaS alone: **you don't need to write a backend.**

```typescript
// No need to write API routes — query the database directly from the frontend
const { data: invoices, error } = await supabase
  .from('invoices')
  .select(`
    *,
    customer:customer_id (name, email),
    items:invoice_items (*)
  `)
  .eq('status', 'overdue')
  .gte('due_date', '2024-01-01')
  .order('due_date', { ascending: false })
```

**What Supabase gives you out of the box:**

| Feature | Supabase Solution | DIY Cost |
|---------|-------------------|----------|
| Database | PostgreSQL (managed) | Buy a server + configure + back up |
| Auth | Auth (email/phone/OAuth) | Implement JWT + session management |
| Storage | S3-compatible object storage | Configure MinIO or go straight to S3 |
| Realtime | WebSocket Realtime | Build your own WebSocket service |
| Edge Functions | Deno Edge Functions | Write Lambda functions |
| API generation | Auto-generate REST API from schema | Hand-write CRUD endpoints |

### Cost Comparison

```
Supabase Free Plan:
  - 500MB database
  - 5GB bandwidth
  - 50,000 monthly active users
  - 2GB storage
  → Great for the MVP stage

Supabase Pro ($25/mo):
  - 8GB database
  - 50GB bandwidth
  - 100,000 monthly active users
  - 100GB storage
  → Good after product validation

Self-hosted PostgreSQL (cheapest option):
  - $5/mo VPS + ops time cost
  - You handle backups, scaling, security yourself
  → Not recommended unless you're a DevOps pro
```

### Other BaaS Options

| Option | Strengths | Weaknesses | Best For |
|--------|-----------|------------|----------|
| Supabase | Feature-rich, open source, PostgreSQL | Free tier limits are tight | Most scenarios |
| PocketBase | Minimalist, single-binary deploy, self-hosted | Fewer features, smaller community | Simple CRUD apps |
| Convex | Great realtime, native TypeScript | Learning curve, platform lock-in | Realtime collaborative apps |
| Appwrite | Feature-rich, self-hosting friendly | So-so performance, smaller community than Supabase | When you need self-hosting |

**My pick: Start with Supabase. If costs balloon after validation, consider migrating to self-hosted Supabase or plain PostgreSQL.**

## Payments: Stripe Is the Standard Answer

### Why Stripe?

For a solo founder, the core requirement for payments is: **don't handle compliance, taxes, or refunds yourself.**

```typescript
// Stripe Checkout — launch a payment page in one call
import Stripe from 'stripe'

const stripe = new Stripe(process.env.STRIPE_SECRET_KEY)

export async function createCheckoutSession(priceId: string) {
  const session = await stripe.checkout.sessions.create({
    mode: 'subscription',
    line_items: [{ price: priceId, quantity: 1 }],
    success_url: `${process.env.NEXT_PUBLIC_URL}/dashboard`,
    cancel_url: `${process.env.NEXT_PUBLIC_URL}/pricing`,
    customer_email: user.email,
  })

  return { url: session.url }
}
```

**Stripe's killer features:**

1. **Hosted checkout page** — Checkout mode, no need to build your own payment UI
2. **Subscription management** — Auto-renew, upgrades/downgrades, cancellations
3. **Tax handling** — Stripe Tax calculates and files automatically
4. **Invoicing** — Auto-generate and send invoices
5. **Global payments** — 135+ currencies supported

### Other Payment Options

| Option | Rate | Strengths | Weaknesses |
|--------|------|-----------|------------|
| Stripe | 2.9% + $0.30 | Most features, best docs | Hard to register from China |
| LemonSqueezy | 5% + $0.50 | PayPal support, friendly to non-US users | Higher rate |
| Paddle | 5% + $0.50 | Handles global taxes, supports Chinese users | Requires review, highest rate |
| Alipay/WeChat Pay | 0.6%–1% | Low rate, China-friendly | China-only, no subscription management |

**My advice: Use Stripe for a global audience. Add LemonSqueezy or Paddle as a fallback for China-based users.**

## Deployment: Vercel Is the Hassle-Free Choice

### Vercel's All-in-One Solution

```bash
# Deploy a Next.js project to Vercel
vercel --prod

# Set environment variables
vercel env add DATABASE_URL

# Check deployment logs
vercel logs --prod
```

**Vercel's killer features:**

1. **Git integration** — Push to main, auto-deploy
2. **Preview Deployments** — Every PR gets a preview URL
3. **Edge Network** — Global CDN, zero config
4. **Serverless Functions** — Auto-scale, pay per invocation
5. **Analytics** — Built-in Web Analytics and Speed Insights

### Cost Comparison

```
Vercel Free:
  - 100GB bandwidth/mo
  - 6,000 build minutes/mo
  - 10 Serverless Functions
  - Unlimited Preview Deployments
  → Enough for the MVP stage

Vercel Pro ($20/mo):
  - 1TB bandwidth/mo
  - 6,000 build minutes/mo
  - 100 Serverless Functions
  - Team collaboration features
  → Upgrade after product validation

Other platforms:
  - Railway: starting at $5, pay by resource
  - Fly.io: starting at $2.5/mo, pay per instance
  - DIY VPS: $5–10/mo, requires ops work
```

**My pick: Start on Vercel Free → upgrade to Pro when traffic picks up → migrate to a self-hosted setup if costs get out of hand.**

## Mistakes I Made

### Mistake 1: Premature Optimization

**What I did wrong:** Started with a microservices architecture, splitting the user system, payment system, and notification system into three separate services.

**What happened:** Spent 3 weeks building infrastructure. In week 4, the requirements changed and I had to rewrite all the inter-service communication logic.

**Lesson:** A one-person company doesn't need microservices. A monolith with clean module boundaries is enough. Reconsider splitting when you have 3+ developers.

### Mistake 2: Picking a Niche ORM

**What I did wrong:** Chose an ORM with only 500 GitHub stars because it was "more performant."

**What happened:** Nobody answered my bug reports. Docs were incomplete. Spent 2 days migrating to Prisma.

**Lesson:** For a solo founder, **community activity is 10x more important than raw performance.** Use mainstream ORMs like Prisma or Drizzle — when you hit issues, you can Google the answer.

### Mistake 3: Self-Hosting an Email Server

**What I did wrong:** To save money, I set up an email server with Postfix.

**What happened:** Major email providers flagged my emails as spam. Users never received their signup confirmations. Churn hit 40%.

**Lesson:** Always use a third-party email service. Pick Resend, SendGrid, or Mailgun — any of them works. $20/mo buys you 99%+ deliverability.

## Summary

There's no silver bullet for a solo founder's tech stack, but there's a clear decision framework:

1. **MVP stage:** Validate your idea with the least money and the most speed. Next.js + Supabase + Vercel + Stripe, ~$1/mo.
2. **Growth stage:** After validation, upgrade services incrementally. Keep monthly costs in the $50–$100 range.
3. **Scale stage:** Once you have revenue, then think about migration and optimization.

**Remember: your goal is to build a product people will pay for, not a perfect tech architecture.**

Your tech stack is a tool, not the goal. Pick the tools that let you ship value fastest, and spend the time you save on understanding users and polishing the product.
