## Intro: A Remote Worker's Philosophy on AI Tools

As a remote developer, your time is your money.

No one's going to walk over to your desk when you're stuck. No whiteboard to sketch architectures on. No colleague nearby for a quick code review.

That's why **remote developers need AI tools more than office-based developers.**

This post shares the 10 AI tools I use every day as a remote/indie developer. **All of them have been battle-tested for at least 3 months.**

## 1. Code Generation & Assistance

### 1. Cursor — AI-first IDE

**Best for:** Daily coding, code refactoring, cross-file edits

**Why I chose it:**
- Not a VS Code plugin, but a **native AI IDE**
- Tab completion accuracy is incredibly high, rarely needs corrections
- Supports cross-file context understanding (@file references)
- Composer mode: generate multiple files at once

**How I use it:**
```
New components: Open Composer, describe requirements, AI generates skeleton
Editing code: Select code, Cmd+K, type your modification intent
Debugging: Select the error, Cmd+L, let AI analyze
```

**Price:** $20/month (Pro), free tier is plenty usable

### 2. Claude / ChatGPT — General-Purpose AI Assistants

**Best for:** Solution design, tech stack decisions, code review

**Why I chose them:**
- Large context windows (Claude 200K / GPT 128K)
- Great for reviewing large codebases
- Strong reasoning ability, ideal for complex problem analysis

**How I use them:**
```
Solution design: Paste requirements doc, let AI analyze
Code Review: Paste PR diff, let AI do first-pass review
Tech selection: Describe the scenario, let AI compare options
```

## 2. Debugging & Troubleshooting

### 3. Warp — AI Terminal

**Best for:** CLI operations, log analysis, Git operations

**Why I chose it:**
- Native AI terminal, natural language to commands
- Built-in AI error analysis
- Smart autocompletion (commands, paths, Git branches)

**How I use it:**
```
"Find files modified in the last 3 days that contain 'TODO'"
"Analyze this error log and find the root cause"
"Write a git command: undo the last 3 commits but keep changes"
```

**Price:** Free

### 4. Sentry + AI — Error Monitoring

**Best for:** Production error investigation

**AI Features:**
- Auto-analyzes error stacks
- Suggests fixes
- Links to related code

## 3. Documentation & Knowledge Management

### 5. Notion AI — Document Assistant

**Best for:** Technical docs, API docs, project wikis

**How I use it:**
```
Technical proposals: Write outline first, AI expands it
API docs: Paste code, AI auto-generates documentation
Weekly reports: List bullet points, AI polishes them
```

### 6. Obsidian + Copilot — Personal Knowledge Base

**Best for:** Technical notes, learning logs, blog drafts

**Why I chose it:**
- Local storage, privacy-safe
- Bidirectional links, knowledge graph
- Rich AI plugin ecosystem

## 4. Design & Prototyping

### 7. v0.dev — AI-Generated UI

**Best for:** Quick prototyping, landing pages, component design

**Why I chose it:**
- Natural language description → directly generates React/Tailwind code
- Supports iterative modifications
- Generated code can be used in production

**How I use it:**
```
"Create a SaaS pricing page with 3 tiers and comparison feature"
"Redesign my blog homepage, minimal style"
```

### 8. Excalidraw + AI — Architecture Diagrams

**Best for:** System architecture diagrams, flowcharts, whiteboard discussions

## 5. Deployment & Operations

### 9. Vercel AI — Deployment Assistant

**Best for:** Frontend project deployment, CI/CD configuration

**AI Features:**
- Auto-detects framework
- Optimizes build configuration
- Analyzes deployment errors

### 10. GitHub Copilot for PR — AI Code Review

**Best for:** Open source PR reviews, team collaboration

**Features:**
- Auto-generates PR descriptions
- AI code review
- Auto-fix suggestions

## 7. AI Usage Principles for Remote Workers

### Principle 1: AI Is Your "Junior Engineer"

Treat AI like a **junior engineer who's always online, never tired, but needs your review**. It handles the grunt work, but decisions and oversight are on you.

### Principle 2: Build Your Own Prompt Library

Save your commonly used prompt templates. Reuse them for similar problems—**don't start from scratch every time.**

### Principle 3: Review Your Toolchain Regularly

AI tools iterate incredibly fast. **Review your toolchain every quarter**—drop what's not working, try what's new.

### Principle 4: Mind Your Data Security

Remote work often involves sensitive code. **Never paste API keys, passwords, or client data into AI tools.**
