## You're Only Using 10% of PowerToys

In the last article, we used PowerToys Keyboard Manager to swap the Ctrl and Win keys. You've got it installed — but if you stop there, you're only scratching the surface.

PowerToys currently has **20+ modules**, and at least 6 of them are daily essentials for front-end developers. The problem is: most people don't know what they can do, or enable them without knowing how to configure them properly.

This article isn't a feature list translation. I'm going to start from **real development scenarios**, telling you what problem each module solves, how to configure it, and how your workflow changes after setting it up.

## 6 Must-Have Modules for Front-End Developers

### 1. FancyZones: Say Goodbye to Manual Window Dragging

**Have you experienced this pain?**

VS Code on the left, Chrome DevTools on the right — manually dragging windows to align, off by 2 pixels. Then open the terminal, gotta drag again. Open Figma too, manually arranging three windows... 5 minutes gone.

FancyZones is a window tiling manager for Windows. Think Magnet / Rectangle on macOS, but **free and more powerful**.

**Core operations**:
- `Win+~` opens the layout editor
- Hold `Shift` and drag a window to the target zone
- `Win+←/→` for quick left/right split

**My layout setup**:

```
┌─────────┬──────────────────┐
│         │                  │
│ Terminal│    VS Code       │
│  25%    │      50%         │
│         │                  │
│         ├──────────────────┤
│         │   Chrome DevTools│
│         │      25%        │
└─────────┴──────────────────┘
```

**Configuration steps**:
1. Open FancyZones → Layout editor
2. Select "Custom" → create a custom layout
3. Set the number of zones and their proportions
4. Check "Override Windows Snap" to replace the system snap behavior

**FancyZones vs macOS Tools Comparison**:

| Feature | FancyZones | Magnet (macOS) | Rectangle (macOS) |
|---------|-----------|-----------------|-------------------|
| Price | Free | $4.99 | Free |
| Custom layouts | ✅ Any partition | ❌ Preset layouts | ❌ Preset layouts |
| Shortcut binding | ✅ Per zone | ✅ | ✅ |
| Multi-monitor | ✅ | ✅ | ✅ |
| Cross virtual desktops | ✅ | ❌ | ❌ |

### 2. PowerToys Run: Alfred for Windows

`Alt+Space` to summon, type anything, hit Enter to execute — that's Run.

If you've used Alfred or Raycast on macOS, you know the feeling. If you haven't, try it once and you won't go back.

**My common search scenarios**:

```
# Search files
docs component → directly opens Component.tsx

# Run calculations
= 1024 * 768 → 786432

# Search system commands
env → opens Environment Variables settings

# Search processes
node → kill a stuck Node process

# Run Winget directly
winget install → search and install apps
```

**Key configuration**:
1. Open PowerToys Run → set shortcut to `Alt+Space` (default)
2. Enable "Search in Web" plugin
3. Enable "Windows Settings" plugin — jump directly to any settings page
4. Enable "Registry" plugin — search the registry (useful for debugging)

**Run vs macOS Tools Comparison**:

| Feature | PowerToys Run | Alfred | Raycast |
|---------|-------------|--------|---------|
| Price | Free | Free (Powerpack $34) | Free (Pro $8/mo) |
| File search | ✅ Windows Search | ✅ Spotlight | ✅ |
| Calculator | ✅ Built-in | ✅ | ✅ |
| Plugin system | ✅ Limited | ✅ Rich | ✅ Rich |
| Clipboard History | ✅ Built-in | ❌ Requires Powerpack | ✅ |
| Window Management | ❌ | ❌ | ✅ |

### 3. AlwaysOnTop: Pin Windows with One Shortcut

`Win+Ctrl+T` keeps the current window always on top.

**Developer scenarios**:
- Pin docs/design specs on top, code below — no more Alt+Tab switching
- Pin the terminal on top for easy log monitoring
- Pin Postman on top while writing API code

No configuration needed — it works out of the box. The only setting is the custom shortcut.

### 4. Color Picker: The Front-End Developer's Eyedropper

`Win+Shift+C` brings up the color picker — click anywhere on screen to grab the color value.

**Output formats**:
```
HEX: #1A1A2E
RGB: rgb(26, 26, 46)
HSL: hsl(240, 28%, 14%)
```

**Configuration tips**:
- Set the default format to HEX (most commonly used in front-end)
- Enable "Color Picker always on top" — won't be obscured by other windows while picking
- Enable "Show color name" — displays the approximate color name

**vs Dedicated Color Tools**:

| Tool | Advantage | Disadvantage |
|------|-----------|-------------|
| PowerToys Color Picker | Free, lightweight, integrated | Limited formats |
| ColorZilla (Chrome) | In-browser picking | Browser only |
| sip (macOS) | Feature-rich | $9.99 |

### 5. File Locksmith: Who's Locking My File?

```
Cannot delete xxx.js: This file is opened in another program.
```

When you hit this error, the traditional approach is: close all programs → still doesn't work → reboot → still errors → force delete.

File Locksmith solves it in one click: right-click the file → "Unlock with File Locksmith" → see which process is holding it → kill that process directly.

**Use cases**:
- `npm run dev` is stuck, can't delete node_modules
- VS Code is locking a file and won't let you delete it
- Excel is locking a CSV file and won't let you overwrite it

### 6. Peek: Quick Preview Without Opening

Select a file → `Ctrl+Space` → a preview window pops up.

Supported file types: images, Markdown, code files, PDFs, text, video.

**Scenario**: Finding a specific screenshot among 10 `screenshot-*.png` files — no need to open them one by one. Peek gives you instant previews, navigate with left/right arrows.

## Advanced Techniques

### Keyboard Manager Does More Than Swap Keys

In the previous article we only used "Remap a key", but Keyboard Manager also has **"Remap a shortcut"** functionality.

**Practical shortcut mappings**:

```json
{
  "remapShortcuts": {
    "global": [
      {
        "originalKeys": "18;9",
        "newRemapKeys": "91;9",
        "operationType": 0,
        "secondKeyOfChord": 0
      }
    ],
    "appSpecific": [
      {
        "originalKeys": "162;192",
        "newRemapKeys": "162;69",
        "targetApp": "Code.exe"
      }
    ]
  }
}
```

What these two mappings do:
- Global: `Alt+Tab` (VK 18=Alt, 9=Tab) → `Win+Tab` (VK 91=Win), using the Win key to switch apps (closer to macOS's Cmd+Tab)
- In VS Code: `Ctrl+`` → `Ctrl+E`, unifying the quick file open shortcut

### The Power of App-Specific Mappings

`appSpecific` lets you customize shortcuts for different applications. This is especially useful for cross-platform developers:

| App | Mapping | Notes |
|-----|---------|-------|
| VS Code | `Ctrl+`` → `Ctrl+E` | Unified file navigation |
| Chrome | `Ctrl+L` → `F6` | Unified address bar focus |
| Terminal | `Ctrl+Shift+T` → `Ctrl+T` | Unified new tab |
| Figma | `Ctrl+Shift+K` → `Ctrl+V` | Paste replacement |

### Advanced Paste

`Ctrl+Shift+V` to summon, supports:
- Plain text paste (strips formatting)
- Markdown → HTML conversion
- Custom paste transformation scripts

**Scenario**: Copying code from a webpage into VS Code — `Ctrl+Shift+V` select "Paste as plain text" and it won't bring in HTML formatting.

Here are the key settings from my actual `settings.json` for reference:

```json
{
  "enabled": {
    "FancyZones": true,
    "Keyboard Manager": true,
    "PowerToys Run": true,
    "AlwaysOnTop": true,
    "ColorPicker": true,
    "File Locksmith": true,

    "Awake": true,
    "Image Resizer": true,
    "Shortcut Guide": true,
    "AdvancedPaste": true,
    "CmdPal": true
  }

```

**Modules I disable** (no reason to enable):
- `MouseWithoutBorders`: needs multiple computers to be useful
- `EnvironmentVariables`: the system settings work fine
- `Hosts`: editing the hosts file directly is more intuitive
- `RegistryPreview`: RegEdit is sufficient

## What PowerToys Isn't Good At

To be fair, PowerToys has its limitations:

| Need | PowerToys Capability | Better Alternative |
|------|---------------------|-------------------|
| Complex keyboard macros | ❌ No loops/conditionals | AutoHotkey |
| Cross-platform key sync | ❌ Windows only | QMK firmware |
| Automated workflows | ❌ Not supported | AutoHotkey / Python |
| Multi-computer clipboard sharing | ⚠️ MouseWithoutBorders is limited | Cross-platform tools like 1Clipboard |
| Window rule automation | ❌ Not supported | AutoHotkey / Komorebi |
